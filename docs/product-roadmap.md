# Kelimelik AI Product Roadmap

## Working product name

Kelimelik AI

Note: "Kelimelik" already appears to be used by an existing Turkish word game app. For now, this project can use "Kelimelik AI" as a working/internal product name. Before App Store / Google Play release, the final public name should be checked for trademark and store-listing risk.

## Core promise

Ezberleme. AI ile kullanmayı öğren.

English alternative:

Don't memorize words. Learn how to use them with AI.

## Product philosophy

The app should feel like a premium, calm, AI-powered vocabulary coach.

Not childish.
Not academic-heavy.
Not a raw database.
Not a basic MVP.

The user should feel:

- I know what to study today.
- AI helps me understand and use words.
- Sets are my study collections.
- Words/Library is my safe archive.
- Practice feels focused and polished.

## Information architecture

### Main tabs

- Today: start studying
- Sets: study collections
- Words: all saved words, search, filters

### Full-screen flows

- Card Sort: main study flow
- Review: quiz/practice flow
- Word Detail: word profile and AI lesson detail
- Profile/Settings: account, language, password, sign out

## Required product features

### Auth

- Login
- Sign up
- Forgot password
- Reset password
- Change password
- Sign out

### Profile / Settings

- User email
- Language selection: Türkçe / English
- Change password
- Sign out
- App version
- Privacy / data note
- Later: delete account

### Localization

Supported app languages:

- Turkish
- English

The language system should not be an afterthought. UI copy should be centralized gradually.

### Core learning

- Add word
- Add + Generate AI
- AI content cached in Supabase
- Sets as study collections
- Words as all saved words archive
- Card Sort
- Meaning Quiz
- Reverse Quiz
- Fill in the Blank
- Word Detail without practice clutter

## Design direction

Calm premium study app.

Visual qualities:

- Clean white/soft surfaces
- Strong but not childish blue
- Clear hierarchy
- Large touch targets
- Polished empty states
- Consistent cards, buttons, inputs, pills
- Minimal technical language
- AI value visible but not gimmicky
